Git Commands (Basics)

Push ( to Existing Repo )

  1. In Terminal, check git status.
  2. Add to Staging Level.
  3. Commit files.
  4. Check Folder names (local's and Github's).
  5. Check current Repo urls (local's and Github's).
  6. Make sure they are matched.
  7. Push to GitHub.
Relate To Description Git Command + hit "Enter"
Directories / Move Around Folders Print Current Directory pwd
Change Directory cd + "Drag the target folder into Terminal on the cursor"
Change Directory to parent folder cd ..
Make Directory / New Folder mkdir "folderName"
Setup Your Details Set up your name globally git config --global user.name "YourName"
Set up your email globally git config --global user.email "yourEmail@gmail.com"
Git Current commit status git status
Initialize git git init
Revert to previous version / Undo git checkout -- .
Files Create a new file touch "index.html"
Staging Add to Staging level git add "index.html"
Add all updated files to Staging level git add -A
Commit Make a commit / Save changes git commit -m 'Add your description of changes here.'

Example...
git commit -m 'Changed font size to 25px.'
Clone Clone an existing repo.
* You may have to enter your username / password / token. 		git clone targetUrl

Example..
git clone https://github.com/LearnWebCode/travel-site-file.git
Server Address Check current repo server address
* Must Navigate to target directory / folder first. 		git remote -v
Change repo server address
* Get target HTTPS on GitHub first. 		git remote set-url origin targetUrl

Example...
git remote set-url origin https://github.com/LearnWebCode/travel-site.git
Push Push repo to GitHub
* Make sure the repo server is matched.
* You may have to enter your username / password / token. 		git push origin master
* Reload GitHub page to check.
